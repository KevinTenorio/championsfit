---
name: to-card
description: Turn the current conversation context into a card description for humans and save it in a .md file. Use when user wants to create a card description from the current context.
---

Resuma em alguns parágrafos o que precisa ser feito (não a nível de implementação, mas a nível de requisitos de negócio) de uma forma legível para humanos que possam querer implementar isso sem o auxílio de agentes de IA.

Sempre que possível, use a linguagem do domínio do projeto (ver o glossário de domínio) para descrever os requisitos. Se o contexto incluir decisões de design ou detalhes de implementação, inclua-os apenas se forem essenciais para entender os requisitos de negócio. Caso contrário, deixe-os de fora para manter o foco nos requisitos.

Use bullet points para destacar requisitos específicos (listagens), mas mantenha a maior parte do texto em formato de parágrafo para facilitar a leitura.

O texto deve ser em português, mas use termos técnicos em inglês quando for o mais usual ou quando eles fizerem parte do vocabulário do domínio do projeto.

O leitor é um desenvolvedor de software com experiência na área, que entende do domínio do projeto, mas não tem conhecimento prévio sobre o contexto específico da conversa. O objetivo é fornecer a ele uma visão clara e concisa do que precisa ser feito, para que ele possa começar a trabalhar na implementação sem precisar de mais esclarecimentos.
